# rusa — Озвучка фильмов и подкастов русскими голосами

**rusa** — это утилита, которая добавляет в вашу коллекцию видео **полноценный закадровый перевод (voiceover)** за одну команду. Просто укажите файл — rusa сам извлечёт субтитры, сгенерирует речь нейросетевыми голосами Microsoft Edge TTS и наложит поверх оригинала.

Подходит для:
- 🎬 **Фильмов и сериалов** — озвучка на русском по имеющимся субтитрам
- 🎙️ **Подкастов и лекций** — закадровый перевод на любой язык

```bash
# Одна команда — готовый результат
rusa movie.mkv
```

## Возможности

- 🎯 **Одна команда** — rusa сам находит субтитры в файле фильма, озвучивает и собирает готовый файл
- 🔊 **Натуральные голоса** — нейронные голоса Microsoft Edge TTS (Светлана и Дмитрий, русские)
- ✂️ **Умная обрезка тишины** — тишина по краям удаляется, но речь и паузы между фразами остаются нетронутыми
- 🧩 **Интеллектуальная сборка** — если реплики накладываются друг на друга, rusa сдвигает их, не обрезая ни слова
- 🌐 **Автоопределение языка** — rusa сам выбирает голос под язык субтитров. Поддерживаются русский, английский, немецкий, французский, испанский, итальянский, португальский, японский, корейский, китайский, арабский, турецкий, нидерландский, польский, шведский, датский, финский, норвежский, чешский, венгерский и иврит. Остальные языки — через явный `--voice`.
- 🎚️ **Регулируемая громкость** — оригинал можно сделать тише (фоном), а перевод — громче (основной звук), или наоборот. Это позволяет настроить баланс так, чтобы голос перевода был чётко слышен, не заглушая оригинал полностью.
- 🔄 **Нормализация** — выровненная громкость по всему фильму
- 🎧 **Любой аудиокодек** — AAC, MP3, Opus, AC3 — на ваш выбор
- ⚡ **Параллельная WAV-обработка** — конвертация `MP3 -> WAV + atempo + trim` выполняется в несколько потоков и обычно заметно быстрее на больших файлах

## Быстрый старт

**Linux / macOS:**

```bash
# Установка
pip install edge-tts tqdm langdetect

# Убедитесь, что ffmpeg установлен
# Linux: apt install ffmpeg  (или brew install ffmpeg на macOS)

# Озвучка
rusa movie.mkv
```

**Windows (PowerShell):**

```powershell
# Установка
pip install edge-tts tqdm langdetect

# Убедитесь, что ffmpeg.exe доступен в PATH (скачайте с ffmpeg.org)

# Озвучка
python rusa.py movie.mkv
```

Готовый файл появится рядом с исходным: `movie_dubbed.mkv`.

Зависимости: `ffmpeg` + `ffprobe` (в составе ffmpeg), `python3`.
`langdetect` нужен для автоопределения языка по содержимому `.srt`.

## Параметры

| Параметр                    | Описание                              | По умолчанию                       |
| --------------------------- | ------------------------------------- | ---------------------------------- |
| `video`                     | Видеофайл (.mkv, .mp4, .avi, …)       | обязательный                       |
| `-o, --output FILE`         | Выходной файл                         | `<видео>_dubbed.mkv`               |
| `-s, --srt FILE`            | Файл субтитров .srt                   | извлечение из видео / поиск рядом  |
| `--voice VOICE`             | Голос edge-tts                        | автоопределение по языку субтитров |
| `--lang LANG`               | Язык субтитров (код ISO 639-1: ru, en, he, de, fr, …) | из голоса `--voice` / авто |
| `--speed SPEED`             | Темп речи TTS                         | `1.5`                              |
| `--orig-vol VOL`            | Громкость оригинала (0.0–1.0)         | `0.65`                             |
| `--tts-vol VOL`             | Громкость TTS (0.0–1.0)               | `0.93`                             |
| `--sync`                    | Синхронизировать субтитры через alass | выкл                               |
| `--keep-temp`               | Не удалять временные файлы            | выкл                               |
| `--threads N`               | Количество потоков TTS                | `6`                                |
| `--aac [BITRATE]`           | Кодек AAC (128, 192, …)               | выкл                               |
| `--mp3 [BITRATE]`           | Кодек MP3 (128, 192, …)               | выкл                               |
| `--opus [BITRATE]`          | Кодек Opus (64, 96, …)                | выкл (по умолч. 64k)               |
| `--ac3 [BITRATE]`           | Кодек AC3 (448, 640, …)               | выкл                               |
| `--from N`                  | Начальный номер субтитра              | все                                |
| `--to N`                    | Конечный номер субтитра               | все                                |
| `--audio-only`              | Только аудио (без видео)              | выкл                               |
| `--normalize [{fast,fine}]` | Нормализация громкости                | выкл                               |

## Использование

```bash
# Базовое использование
rusa movie.mkv

# Мужской голос + ускорение
rusa --voice ru-RU-DmitryNeural --speed 1.8 movie.mkv

# Ивритские субтитры + голос Хилы (автоопределение)
rusa --lang he movie.mkv

# Иврит явно: голос + язык
rusa --voice he-IL-HilaNeural --lang he movie.mkv

# Свои субтитры, темп
rusa -s subs.srt --speed 2.0 movie.mkv

# AAC с нормализацией качества, только аудио
rusa --aac 192 --normalize fine --audio-only movie.mkv

# Opus 96kbps + нормализация
rusa --opus 96 --normalize movie.mkv

# Спектр субтитров (первые 30), MP3
rusa --from 1 --to 30 --mp3 128 movie.mkv

# Только аудио, диапазон
rusa --from 10 --to 50 --audio-only movie.mkv

# Список доступных голосов
rusa --voice

# Русские субтитры из внешнего файла
rusa -s subtitles.ru.srt movie.mkv

# Свои субтитры + синхронизация + Opus 96kbps
rusa -s subs.srt --sync --opus 96 movie.mkv
```

## Голоса

Актуальные русские голоса Microsoft Edge TTS:

| Голос                  | Качество                       |
| ---------------------- | ------------------------------ |
| `ru-RU-SvetlanaNeural` | Женский, нейронный (умолчание) |
| `ru-RU-DmitryNeural`   | Мужской, нейронный             |

Поддерживается **более 80 языков** — от английского и немецкого до японского и иврита. Полный список голосов: `rusa --voice`.

Язык определяется автоматически: по расширению файла (`.ru.srt`, `.en.srt`, …) или через анализ содержимого субтитров (библиотека `langdetect`).
Если вы используете `--lang`, rusa ищет именно субтитры этого языка и не подставляет произвольный соседний `movie.srt`.
Для языков, которых нет в карте автоопределения, укажите голос явно через `--voice`.

## Автоопределение языка

Приоритет определения голоса:

1. Явный `--voice` (если указан)
2. Расширение файла субтитров: `movie.ru.srt` → `ru-RU-SvetlanaNeural`
3. Анализ содержимого субтитров через `langdetect` (если установлен)
4. Если ничего не помогло — rusa завершится с просьбой указать `--voice` явно

### `--lang` и явный голос

- `--lang he` означает: искать ивритские субтитры и, если язык поддержан картой rusa, выбрать голос автоматически.
- Если язык не поддержан картой авто-голосов, rusa завершится с понятной ошибкой и попросит указать `--voice`.
- Комбинация `--lang xx --voice some-VoiceNeural` допустима: `--lang` управляет поиском субтитров и metadata выходного трека, `--voice` управляет озвучкой.

## Как это работает

1. **Субтитры** — извлекаются из видео или берутся из внешнего `.srt`-файла переданного в командной строке
2. **Синхронизация** (опционально) — сначала субтитры синхронизируются (через alass) с оригинальным саундтреком
3. **TTS** — каждый субтитр генерируется Microsoft Edge TTS, сплит длинных фраз (>3000 символов) по знакам препинания
4. **Конвертация** — atempo + обрезка лидирующей и конечной тишины через areverse (два прохода start_periods=1 с разворотом).
5. **Сборка** — каждый сегмент пишется в свой таймкод. При перекрытии — каскадный сдвиг вперёд (target = max(cur_frame, sf)), без обрезания речи.
6. **Микс + mux** — оригинал + voiceover в один файл. Если исходные субтитры нельзя скопировать в выходной контейнер как есть, rusa пробует перекодировать их в SRT; если и это не удалось, сохраняет результат без субтитров, но не роняет весь процесс.

## Диагностика

### Ошибка `Subtitle codec mov_text ... is not supported`

Это происходит, когда исходный контейнер содержит текстовые субтитры в формате, который нельзя просто скопировать в выходной `.mkv` один-в-один.
Текущая логика rusa сначала пытается сделать `-c:s copy`, затем автоматически повторяет mux с перекодированием субтитров в `SRT`.
Если и это не удалось, rusa предупреждает и сохраняет выходной файл без субтитров.

## Логика сборки voiceover

Сборка голосового перевода работает по принципу **каскадного смещения**:

1. **Обрезка тишины** — у каждого WAV-файла обрезается **лидирующая и конечная тишина**
   через `areverse`-трюк: `silenceremove=start_periods=1` → `areverse` →
   `silenceremove=start_periods=1` → `areverse`. Надёжно отделяет тишину от речи
   (в отличие от `stop_periods`, который срезает межфразовые паузы).
2. **Точная вставка** — каждый сегмент пишется в позицию, соответствующую его таймкоду
   из субтитров (`start_ms`).
3. **Каскадный сдвиг** — если предыдущий сегмент ещё не закончился (его данные выходят за
   границу `start_ms` следующего), следующий сегмент **сдвигается вперёд** на освободившееся
   место. Это поведение реализовано через `target = max(cur_frame, sf)`.
4. **Если зазора нет** — сегмент вставляется ровно в свой таймкод, без задержки.

**Когда реплики не успевают за своим временем — они штабелируются (сдвигаются).  
Когда успевают — вставляются точно по времени.**

### Почему не «обрезание по end_ms»

Сегменты НЕ обрезаются по `end_ms` субтитра. Если обрезать — часть слова может быть
отсечена (закадровый голос должен звучать естественно). Вместо обрезания используется
каскадный сдвиг: сегмент, не умещающийся в свой таймслот, просто сдвигается вперёд,
сохраняя целостность произнесённого текста.

### Алгоритм сборки (псевдокод)

```
cur_frame = 0
for each segment in sorted(segments):
    sf = segment.start_ms * sample_rate / 1000  // таймкод из SRT
    target = max(cur_frame, sf)                 // сдвиг, если не успеваем
    
    write segment.audio at position target      // запись в WAV
    
    cur_frame = target + segment.duration_in_frames  // следующий стартует отсюда
```

### Когда возникает сдвиг

Если subtitle A имеет длительность 2000ms, а subtitle B начинается через 1500ms,
но TTS-аудио A после ускорения и обрезки тишины занимает 1800ms:
```
A: |———— речь (1800ms) ————|
B:               |———— речь ————|
                  ↑ B должен быть здесь (1500ms), но A ещё не кончился
                  ↓ B сдвигается
A: |———— речь (1800ms) ————|
B:                         |———— речь ————|
                            B начинает с 1800ms, а не с 1500ms
```
B слышен с задержкой, но зато **целиком**, без обрезания слов.

## Кэширование

TTS-кэш включён:

- готовые MP3 от `edge-tts` кэшируются по паре `voice + text`
- повторный запуск с теми же репликами и тем же голосом переиспользует кэш и пропускает сетевой вызов TTS
- по умолчанию кэш хранится в `~/.cache/rusa/tts`
- путь можно переопределить через `RUSA_CACHE_DIR`

WAV-кэш пока не используется: этап `MP3 -> WAV` просто ускорен за счёт параллельной обработки.
